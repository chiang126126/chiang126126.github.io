## 三、异常信号识别

> 满足**任一**条件入选"异常币候选"（初筛 5-10 个/次）：

### A1：OI 24h 大幅变动

- **条件**：OI 24h 变化 > ±20%
- **方向含义**：
  - OI 急增 + 价格上涨 = 新多头入场（可能是主升浪启动）
  - OI 急增 + 价格下跌 = 空头加仓（可能是做空机会）
  - OI 急减 = 大规模平仓（可能是行情结束或爆仓）
- **22 维**：维度 2（OI）/ 维度 9（Binance OI 占比）
- **加分项**：（待 P1 填充）

### A2：资金费率异常

- **条件**：|funding rate| > 0.05%（即 |0.0005|）
- **方向含义**：
  - 高正费率（> +0.05%）= 多头拥挤，反向信号（考虑做空）
  - 高负费率（< -0.05%）= 空头拥挤，可能反弹
  - 历史费率持续同向 = 方向确立
- **22 维**：维度 4（各所费率）/ 维度 5（费率历史）
- **加分项**：（待 P1 填充）

### A3：价格与 MA20 大幅偏离

- **条件**：价格偏离 20 日均线 > ±10%
- **方向含义**：
  - 大幅正偏离 = 短期过热，注意回调风险
  - 大幅负偏离 = 超跌，可能反弹
- **22 维**：维度 1（K 线）
- **加分项**：（待 P1 填充）

# 在 src/ai_layer/prompt_builder.py 里
SYSTEM_PROMPT = f"""
你是一个专业的加密货币合约交易分析师，遵循以下框架进行判断：

{open("obsidian-vault/02-Frameworks/cresus-master-framework.md").read()}

对于每一个候选币种，你需要：
1. 判断属于哪种"结构类型"（A/B/C/无）
2. 给出操作方向（做多/做空/不做）
3. 给出信心度（0-100）
4. 列出支持理由（对应 22 维数据）
5. 给出进场价位 / 止损价位建议（可选）

以 JSON 格式输出。
"""
