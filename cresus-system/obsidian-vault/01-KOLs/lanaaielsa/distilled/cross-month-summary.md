### 框架二：车头扎堆时点复盘法（可复制方法论）

```
步骤（用 gmgnai + 推特时间线）:

1. 打开旧标的 K 线

3. 找到车头扎堆上车的时间点
4. 去推特时间线找该时点流量最高的帖子
5. 分析扎堆原因 → 车头带头 / 叙事升级 / 特定事件

信号强度：
  车头 = 带头 → 强信号，可跟随
  叙事升级 → 需验证 + 等待
  随机扎堆 → 弱信号，警惕
```

### 框架三：OTC + 筹码结构神盘机制

```
神盘起飞真实要素：
1. 底部筹码结构合理（控制好）
2. 多波 OTC 资金接力
3. 合适的时机切入

OTC 运作：
  项目方 × OTC 商签约（如 5 万 U 配 1% 筹码）
  ↓
  OTC 商市价买（买不到的部分项目方补足 1%）
  ↓
  OTC 交易 = 真金白银注入池子
  ↓
  打开天花板 → 引起 FOMO → 跟单涌入

启示：
  单看「地址持有量、捆绑比例」= 看不到 OTC 流
  真正的信号来自「OTC 商群体」= 市场的 K 线涡轮
```

### 框架四：跟单聪明钱的两种模式

**模式一：有规律的聪明钱（推荐）**
```
特征：极强纪律性
- 亏损后从固定金额重启
- 大盈利立即提走
- 举例：$hype 聪明钱（每次 300 万重启）

跟单策略：监控该地址所有操作，同步建仓
```

**模式二：大赌狗滚仓（谨慎）**
```
特征：高波动，历史多次 1M → 10M
- 但每次都折在 20-100 万 U
- 举例：$pepe @JamesWynnReal

跟单触发条件：爆仓后或亏回 2 万 U 以内
跟单规模：1 万 U（小仓位试）
```

### 框架五：AI 自动交易策略（3000x 复盘启示）

```
核心理解：
  行情 = 90% 贡献
  策略 = 剩余部分
  AI 的价值 = 排除人性干扰（不扩大杠杆、不 FOMO）

适用场景：
  行情启动的早期识别 + 机械化执行

瓶颈:
  资金规模 > 10 万后杠杆使用问题
  需研究不同规模下的杠杆适配方案

衍生策略：
  Polymarket 5/15 min 邪修策略
  利用 AI 不受情绪影响做高频特定 timeframe
```

### AI 策略使用原则

```
适用:
- 行情启动期的机械化高频
- 特定 timeframe 的套利策略
- 排除人性干扰的纪律执行

不适用:
- 宏观判断 / 叙事预判

核心心态:
- 把 AI 当工具，不是稳定印钞机
- 「邪修」思维：出奇招避开主流路径
```
