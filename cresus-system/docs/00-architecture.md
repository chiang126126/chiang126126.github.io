# 整体架构

## 顶层数据流

```
┌─────────────────────────────────────────────────────────┐
│                    手动研究链（日常）                    │
├─────────────────────────────────────────────────────────┤
│  Twitter / Grok / Claude                                │
│        │                                                │
│        ▼                                                │
│  蒸馏 prompt → Claude Code                              │
│        │                                                │
│        ▼                                                │
│  Obsidian Vault（KOL 原文 + 蒸馏 + 框架卡 + 数据源）     │
└─────────────────────────────────────────────────────────┘
                     │  knowledge source
                     ▼
┌─────────────────────────────────────────────────────────┐
│                    自动扫币链（7×24）                    │
├─────────────────────────────────────────────────────────┤
│  [数据采集层] 40% 代码                                   │
│     • 每 5 分钟扫 100 币基础数据                         │
│     • 基础计算：vol/OI, Binance OI 占比, 价格变化        │
│     • 硬过滤：vol/OI > 20x 直接排除                      │
│     • 挑出异常币 5-10 个                                 │
│         │                                                │
│         ▼                                                │
│  [数据组装层] 40% 代码                                   │
│     • 对每个异常币拉完整 22 维数据                       │
│     • 多周期：日线 7 天 + 4H 6 根 + 1H 6 根              │
│     • 币种背景：市值 / 持仓集中度 / 聪明钱 / KOL         │
│     • 大盘环境：BTC 价格 / 恐惧贪婪                      │
│     • 格式化成结构化文本                                 │
│         │                                                │
│         ▼                                                │
│  [AI 判断层] 60% DeepSeek                                │
│     • system prompt：7 位 KOL 完整框架 (~4K tokens)      │
│     • user prompt：22 维 + 多周期数据 (~6K tokens)       │
│     • 不预设方向，只问：什么结构？机会？方向？           │
│     • 返回：结构类型 / 方向 / 信心度 / 理由              │
│         │                                                │
│         ▼                                                │
│  [执行层] 代码                                           │
│     • 信心度 > 70  → 开仓（接 LiveOMS，可选）            │
│     • 信心度 50-70 → 观察（记录，不开仓）                │
│     • 信心度 < 50  → 不做                                │
│         │                                                │
│         ▼                                                │
│  面板（GitHub Pages）+ Discord 通知                     │
└─────────────────────────────────────────────────────────┘
```

## 进程与部署

| 模块 | 运行位置 | 频率 |
|---|---|---|
| 数据采集 | VPS（或本地 dev） | 每 5 分钟 |
| 数据组装 | VPS | 触发式（有异常币时） |
| AI 判断 | VPS（调用 DeepSeek API） | 触发式 |
| Discord 通知 | VPS | 实时 |
| 面板前端 | GitHub Pages | 静态 + 拉 API |
| 面板后端 | VPS（FastAPI） | 常驻 |

## 密钥管理

- 所有密钥放 `.env`（在 private repo 根目录）
- `.env` 进 `.gitignore`，永远不 commit
- 提供 `.env.example` 作为模板
- 部署到 VPS 时用 `scp` 上传 `.env`，不走 git

## 风险隔离

1. **Exchange API 全部用只读权限**，不授权提现/划转
2. 开仓用**子账户**，主账户资金不暴露
3. P6 阶段严格走：模拟盘 3 天 → mini 仓 1 周 → 正式
4. 必须配置移动止盈止损
